# Supabase

**Supabase** is an open-source backend-as-a-service platform that provides developers with tools to build applications quickly without managing server infrastructure. It is often described as an **open-source alternative to Firebase**, built around **PostgreSQL**.

## Core Features
- **Database:**  
  - Full **Postgres database** for every project.  
  - Includes extensions, backups, and row-level security.  
- **Authentication:**  
  - Supports email/password, passwordless, OAuth, and mobile logins.  
  - Integrated identity providers and APIs.  
- **Storage:**  
  - Store, organize, and serve large files.  
  - Integrated with Postgres and secured with access policies.  
- **Realtime:**  
  - Listen to database changes in real time.  
  - Sync user states across clients and broadcast updates.  
- **Edge Functions:**  
  - Globally distributed server-side functions.  
  - Run code close to users for low latency.  
- **APIs:**  
  - Instant RESTful and GraphQL APIs generated from your database schema.  
- **Vector embeddings:**  
  - Built-in support for AI/ML applications using vector search.

## Developer Experience
- Easy setup with a hosted Postgres instance.  
- Rich documentation and tutorials.  
- CLI tools and dashboard for management.  
- Strong community support and open-source contributions.

## Use Cases
- Building web and mobile apps with authentication and storage.  
- Real-time dashboards and collaborative applications.  
- AI-powered apps using vector search.  
- Rapid prototyping without complex backend setup.

## Significance
- Supabase combines the reliability of **Postgres** with modern developer tools.  
- Offers scalability, security, and flexibility for startups and enterprises.  
- Provides a familiar SQL-based workflow while competing with Firebase’s ease of use.
